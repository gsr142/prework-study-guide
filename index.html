Git

  • The git command must precede every Git statement.
  • The checkout command is to move the working branch to a new branch.
  • The -b flag creates a new branch.
  • We have added the prefix, feature, to indicate that it is a feature. It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch.
  • Always ensure you are working on the correct branch, avoid coding directly on main or master.
  • Create feature branch to maintain main/master, and track changes made via feature branches.
  • A pull request, or PR, is used to merge a comparison branch (in our case, the feature branch) into the base branch.
  • Steps for version control
    1. git status: checks what branch we are currently on
    2. git checkout -b branch-name: creates a new branch and switches to it
    3. git status: verify that you are working on feature branch
    4. Write your code
    5. git add -A: The add command adds modifications in the current working branch to the staging area. -A indicates we want to add all changes. The changes are now staged to be committed.
    6. git commit -m "add code": This command captures a snapshot of the currently staged changes, once committed, the feature branch will be updated with the starter code.
    7. git pull origin main: We need to double check that our local branch is synced with the base branch in github. this command pulls any changes from the base branch, and ensures we are working on updated code.
    8. git push origin feature/starter-code: This command pushes the local changes we have made, to the remote Github branch.
    9. Go to the repository on Github, main brainch will be displayed by default. Select you feature branch from the dropdown, and click on the "Compare and pull request" button.
    10. Give the PR a descriptive title. In the body, add instructions for the team to review the changes, functionality, and testing.
    11. Designate the base branch as "main", and the comparison branch as your feature branch.
    12. Click on the "Create pull request" button.
    13. Once the feature branch has been reviewed and is ready to merge with the main branch, click the "Merge pull request" button.
